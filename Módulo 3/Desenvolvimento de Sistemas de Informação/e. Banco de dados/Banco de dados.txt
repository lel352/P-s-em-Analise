Banco de dados
Nesse módulo foram apresentados os conceitos de banco de dados, com destaque para os bancos de dados relacionais, que são amplamente utilizados e consolidados no mercado. Sequencialmente, discutiu-se as aplicações de banco de dados no contexto acadêmico e de mercado em por fim, foi vista uma nova tecnologia conhecida como NoSQL que rompeu com os modelos tradicionais de banco de dados relacionais.

Banco de dados são coleções organizadas de dados que se relacionam de forma a criar algum sentido (informação) e dar mais eficiência durante uma pesquisa ou estudo científico.

São de vital importância para empresas e há mais de duas décadas se tornaram a principal peça dos sistemas de informação e segurança.

Um banco de dados é um conjunto de grupos de informações que estão relacionadas entre si de alguma maneira (por exemplo, por meio de campos comuns). Os grupos lógicos de informações poderiam incluir categorias como dados do cliente, informações sobre um pedido e dos produtos. Um sistema de gerenciamento de banco de dados (SGBD ou database management system – DBMS) é o software que cria e manipula esses bancos de dados.

• Arquivos: Um arquivo de dados contém uma lista eletrônica de informações, que é formatada para uma determinada transação, e essas informações são modificadas e manipuladas por programas escritos para essas finalidades. Com frequência, os arquivos gerados por sistemas legados antigos estão em um formato exclusivo ou fechado (também denominado formato proprietário), ao passo que os novos sistemas usam um formato padrão como o CSV (comma separated value, em que os dados são separados por um caractere específico, como uma vírgula) ou delimitado por tabulações.

 

• Arquivos de Consultas: Os SGBDs de usuários finais, como o Microsoft Access, fornecem suporte a bancos de dados de pequena escala usados para melhorar a produtividade pessoal, e os SGBD empresariais, como DB2, Jasmine, SQL Server e Oracle, podem gerenciar imensos volumes de dados e suportar aplicativos que controlam toda a empresa. Um SGBD de usuário final é significativamente mais barato e mais fácil para iniciantes que seu congênere empresarial, mas não possui os recursos ou as capacidades necessárias para fornecer suporte a sistemas críticos para o funcionamento da empresa ou de grande escala. Os SGBDs em código aberto também são populares, como o MySQL.

 

Existem vários Modelos de Base de Dados:

• Modelo Plano (ou tabular), 

• Modelo em Rede, 

• Modelo Hierárquico, 

• Modelo Relacional, 

• Orientado a objetos 

• Objeto-Relacional.

 

Nos bancos de dados relacionais as tabelas possuem um número estabelecido de colunas e um número variável de linhas, que contêm ocorrências de dados. A Structured Query Language (SQL) é a linguagem-padrão para o acesso de dados em tabelas e trabalha com tabelas completas, e não com os registros individuais das tabelas.

Sistemas Gerenciadores de Bancos de Dados são usados em muitas aplicações, atravessando virtualmente a gama inteira de software de computador. Os Sistemas Gerenciadores de Bancos de Dados são o método preferido de armazenamento/recuperação de dados/informações para aplicações multiusuárias grandes onde a coordenação entre muitos usuários é necessária.

NoSQL é um movimento que promove soluções de armazenamento de dados não relacionais. Ele é composto por diversas ferramentas que, de forma particular e específica, resolvem problemas como tratamento de grandes volumes de dados, execução de consultas com baixa latência e modelos flexíveis de armazenamento de dados, como documentos XML ou JSON.

As tecnologias NoSQL não têm como objetivo substituir os bancos de dados relacionais, mas apenas propor algumas soluções que em determinados cenários são mais adequadas. Desta forma é possível trabalhar com tecnologias NoSQL e banco de dados relacionais dentro de uma mesma aplicação.

Bancos de dados NoSQL são criados para modelos de dados específicos e têm esquemas flexíveis para a criação de aplicativos modernos. Os bancos de dados NoSQL são amplamente reconhecidos por sua facilidade de desenvolvimento, funcionalidade e performance em escala. Eles usam vários modelos de dados, incluindo documento, gráfico, chave-valor, memória e pesquisa.

Os bancos de dados NoSQL usam diversos modelos para acessar e gerenciar dados, como documento, gráfico, chave-valor, em memória e pesquisa. Esses tipos de banco de dados são otimizados especificamente para aplicativos que exigem modelos de grande volume de dados, baixa latência e flexibilidade. Esses requisitos são atendidos mediante o relaxamento de algumas restrições de consistência de dados dos outros bancos.
